# 今後の実装予定メモ

最終更新: 2026-02-22 21:08 (JST)

このメモは、次期開発候補の要点を簡潔に整理したものです。  
優先度は暫定で、実装着手時に再評価します。

## 直近候補（MVP）

1. 試験日逆算の分散プリセット
目的: 試験日前に必要な復習回数を確保する。  
実装メモ: `examDate` をセット単位で保持し、`nextDue` が試験日を超える場合は前倒し補正する。

2. 自信度入力と誤差可視化
目的: 過信・過小評価を見える化して学習精度を上げる。  
実装メモ: 回答時の自信度と実正答率の差分を、セット単位/タグ単位で可視化する。

3. 弱点トピック自動インターリーブ
目的: 同カテゴリ連打を避けつつ弱点を重点的に強化する。  
実装メモ: タグ別スコア（誤答率・低自信率・期限超過率）で出題キューを重み付き生成する。

4. フィードバック遅延モード
目的: 想起練習の効果を高める。  
実装メモ: 正誤・解説表示タイミングを `即時 / 5問後 / セッション最後` で切替可能にする。

5. デッキ品質チェック（重複/曖昧文）
目的: 問題品質のばらつきを減らす。  
実装メモ: 保存前に重複候補・曖昧表現を検出し、修正候補を表示する。

## 中期候補

6. 復帰ミッション（休眠対策）
目的: 数日離脱したユーザーの復帰率改善。  
実装メモ: 未学習日数に応じて 3日間の短時間ミッションを自動提案する。

7. 学習ヘルススコア
目的: 学習状態を一目で把握し、行動に落とし込む。  
実装メモ: 継続率・期限超過・弱点改善速度などから 0-100 スコアを日次算出する。

8. 問題ごとの判別力ダッシュボード
目的: 良問/要修正問題を識別し、問題集品質を上げる。  
実装メモ: 問題ごとに難易度・判別力指標を集計し、管理画面で確認できるようにする。

## 品質・連携基盤

9. WCAG 2.2 重点監査
目的: キーボード操作・コントラスト・フォーカス可視性を改善する。  
実装メモ: 主要導線を自動/手動で監査し、優先度付き修正リストを運用する。

10. xAPI エクスポートの最小実装
目的: 学習ログを外部分析基盤に連携可能にする。  
実装メモ: `answered / reviewed / completed` の3イベントを JSON 形式でエクスポートする。

## 備考

- 実装時は `src/db.ts` のクラウド/ローカル分岐を必ず両対応にする。
- 復習日 (`nextDue`) は `YYYY-MM-DD` のローカル日付で扱う。
- ユーザーに見える仕様変更時は、`src/pages/ReleaseNotesRoute.tsx` に追記する。
