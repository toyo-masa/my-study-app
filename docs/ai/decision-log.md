### t分布表の両側有意水準列は 0.02 ではなく 0.025 を採用する
- t分布表の `T_TWO_SIDED_ALPHA_COLUMNS` は `[0.2, 0.1, 0.05, 0.025, 0.01]` とする
- 確率列の表示は `toFixed(2)` ではなく、末尾ゼロだけを落とす `formatProbability` を使い、`0.025` を丸めず表示する

## 2026-04-29
### Markdown の数式だけの行は中央寄せにする
- Markdown 内の通常の文中数式は本文の流れを維持し、段落内の要素が KaTeX 数式だけの場合に中央寄せする
- `$$...$$` のブロック数式は `.katex-display` 側でも中央寄せを明示する
- 選択肢内の数式だけの行は、選択肢の視線移動を優先して左寄せへ戻す
- 数式だけの段落判定は CSS の `:has()` ではなく、`MarkdownText` 側で空白以外の子要素を見て `markdown-math-only` class を付ける

### 理由
- 問題解説では計算式だけの行が途中に挟まることが多く、本文と同じ左寄せだと計算ステップとして読み取りにくい
- すべての数式を中央寄せにすると文中の `χ²` や `μ = 0` まで不自然になるため、単独行だけに限定する
- 選択肢はラジオボタンや番号との位置関係が重要で、中央寄せにすると短い数式選択肢の開始位置が揃わず読みにくい
- CSS の `:has(> .katex:first-child:last-child)` はテキストノードを考慮せず、文章と数式が混在する段落まで誤って中央寄せ対象にするため

### 統計分布表パネルのリサイズは縮尺変更ではなくコンテナサイズ変更で扱う
- 問題画面の統計分布表は、パネル右下のハンドルでコンテナサイズだけを調整し、表本体は内部スクロールで読む

### 理由
- 分布表は表の可読性が優先で、縮尺まで落とすとセル値を読み取りにくくなる
- 既存の埋め込み `DistributionTables` は内部スクロール前提の構成なので、コンテナだけ可変にした方が最小差分で済む

### 電卓の `1/x` は新演算子ではなく既存式ラッパーとして扱う

- 空の状態や演算子直後では `1÷(` を挿入し、既存の括弧補完・パーサ・表示ロジックをそのまま使う

### 理由
- 既存の再帰下降パーサは `÷` と括弧で十分に逆数を表現でき、新しい関数トークンを増やす必要がない
- `sqrt` などの「直前の値を包む」操作と同じ流れで実装した方が、既存 UI の一貫性を保ちやすい

### 問題回答画面の統計分布表は既存コンポーネントを埋め込み再利用する
- `DistributionTables` にヘッダー表示・プレビュー表示の切り替えだけを足し、問題回答画面では表のみの埋め込みモードで使う
- 新しい分布表専用コンポーネントは増やさず、ルート画面とツールパネルで同じ表データ定義を共有する

### 理由
- 正規分布表・t 分布表・カイ二乗分布表の中身は既存の `DistributionTables` で完結しており、表だけ欲しい要件は表示切り替えで吸収できる
- データ定義やセル選択ロジックの二重管理を避け、問題画面用の追加差分を最小に保ちやすい

### 手書きメモ欄の自動スクロール先は window ではなく学習画面内スクロールへ寄せる
- 手書きメモ欄リサイズ中の自動スクロールは `window.scrollBy` 前提にせず、学習画面の `.content-area` を優先してスクロールする
- 学習ルートで app shell を fixed 化したあとでも、メモ欄の下方向拡張と iPad のパン抑止を両立できる構成を優先する

### 理由
- remote 側の自動スクロール改善と、local 側の app shell 固定化はそのままではスクロール対象が食い違う
- 実際に動いているスクロールコンテナへ揃えた方が、空白方向のパンを抑えつつリサイズも継続しやすい

### 手書きメモ欄のリサイズ中は root 全体を一時ロックする
- 手書きメモ欄の下端ドラッグ中だけ `html` / `body` に専用クラスを付け、`overflow` / `overscroll` / `touch-action` を止める
- リサイズ追従はハンドル要素内の `onPointerMove` ではなく、window 監視へ寄せて pointer capture と併用する

### 理由
- iPad 系では、リサイズハンドルを下へ引く操作がページ全体のパンや bounce と競合しやすい
- ドラッグ中だけブラウザ既定挙動を止める方が、通常スクロールを壊さずに症状へ直接効く

### 学習画面の app shell 自体を viewport に固定する
- `app-container.study-mode-active` を `position: fixed` + `inset: 0` にして、学習中の殻全体が viewport から外れないようにする
- ヘッダーは flex shrink を止め、戻るボタンやツール導線が常に上端に残る前提を強める

### 理由
- root のスクロールロックだけでは、iPad 系ブラウザで app shell 全体がずれたような状態に入ることがある
- 問題操作より優先して復帰導線を守る必要があり、ヘッダーを含む殻ごと固定した方が安全

### 学習画面では app shell だけでなく document root もスクロールロックする
- `study` / `memorization` ルートでは `app-container` だけでなく `html` / `body` にも専用クラスを付け、root 側の `overflow` と `overscroll` を止める
- 学習画面の通常スクロールは `content-area` に集約し、ブラウザ全体が空白方向へパンする余地を減らす

### 理由
- iPad 系ブラウザでは、内側コンテナを `overflow: hidden` にしていても document 側のパンが残ることがある
- 学習中は問題文・選択肢・手書きメモの操作が中心で、root スクロールを許可する利点より誤操作防止の優先度が高い

### 問題画面電卓の高度計算は独自パーサで扱う
- 問題画面の電卓は `eval` を使わず、括弧・冪乗・平方根・`log`・`ln`・`sin/cos/tan` を扱える再帰下降パーサで式を評価する
- 三角関数は度数法固定とし、`sin(90)=1` になる前提で扱う
- `x²` / `x³` は専用ボタン、一般の冪乗は `xʸ` と `^` で入力する
- 未閉鎖の括弧は表示上は薄い `)` で補助し、評価時は末尾で自動補完する
- バックスペースは文字単位ではなく、`sqrt(` や `sin(`、`^2` / `^3` のような入力単位でまとめて消す
- 式末尾が演算子などで未完成のまま `=` が押された場合はエラーにせず no-op にする
- ボタン配置は科学関数を上段、数字キーを下段、演算子を右列へ寄せた iPad 風レイアウトに寄せる
- 表示上の数値は 3 桁区切りで整形し、内部の評価式にはカンマを入れない
- リサイズ下限は内容量ベースで広めに取り、通常の viewport では全ボタンが枠内に収まる前提を優先する
- `±` は末尾の数値セグメントの符号だけを反転し、`%` は末尾数値を `÷100` に置き換える簡易入力として扱う
- 電卓ウィンドウのリサイズは縦横比固定に切り替え、縮小時は CSS 変数でボタンや文字サイズも比例して縮める
- `√` や `sin` などの前置関数は、直前の値があれば新規に `×` を挟まず、その値を包む形で入力する
- その後の操作性要望に合わせ、電卓のリサイズは再び縦横独立に戻し、縮小時の中身スケールだけを維持する
- さらに調整し、リサイズハンドルは右下のみに限定しつつ、ボタン群は残り高さいっぱいまで広がるレイアウトへ変更する
- 計算結果表示時は iPhone 電卓に寄せて、小さく薄い計算式を結果の上に残す
- 履歴表示行は条件付きで増減させず、最初から高さを確保して表示欄全体の揺れを防ぐ

### ベクトル・行列ラボは front-end only の単独 SVG アプリとして追加する
- ベクトル・行列ラボは API / DB / クラウド同期 / ローカル保存を増やさず、フロントエンドだけで完結する単独アプリとして追加する
