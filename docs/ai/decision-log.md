# Decision Log

## 2026-04-05
### 解答画面ヘッダーの右上操作は `headerActions` で拡張する
- `QuizSessionLayout` の右上領域は AI チャットボタン専用にせず、`headerActions` と既存 AI チャットトグルを並べられる構成にする
- AI チャットトグルの表示条件は既存のまま維持し、ツールボタンは AI パネル表示条件に依存させない
- 解答中の軽量ツールは `SessionToolsLauncher` で一覧・パネル・入力状態をまとめて持つ

### 問題画面電卓の高度計算は独自パーサで扱う
- 問題画面の電卓は `eval` を使わず、括弧・冪乗・平方根・`log`・`ln`・`sin/cos/tan` を扱える再帰下降パーサで式を評価する
- 三角関数は度数法固定とし、`sin(90)=1` になる前提で扱う
- `x²` / `x³` は専用ボタン、一般の冪乗は `xʸ` と `^` で入力する
- 未閉鎖の括弧は表示上は薄い `)` で補助し、評価時は末尾で自動補完する
- バックスペースは文字単位ではなく、`sqrt(` や `sin(`、`^2` / `^3` のような入力単位でまとめて消す
- 式末尾が演算子などで未完成のまま `=` が押された場合はエラーにせず no-op にする
- ボタン配置は科学関数を上段、数字キーを下段、演算子を右列へ寄せた iPad 風レイアウトに寄せる
- 表示上の数値は 3 桁区切りで整形し、内部の評価式にはカンマを入れない
- リサイズ下限は内容量ベースで広めに取り、通常の viewport では全ボタンが枠内に収まる前提を優先する

### 理由
- 学習/暗記の両ルートへ同じ右上 UI を最小差分で差し込める
- 将来ツールを増やしても、AI チャット側の条件分岐を崩さず拡張しやすい
- 問題画面ショートカットとの干渉を、ツール UI ルート単位で管理しやすい
