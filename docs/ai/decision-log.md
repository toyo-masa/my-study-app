# Decision Log

## 2026-04-10
### 学習画面では app shell だけでなく document root もスクロールロックする
- `study` / `memorization` ルートでは `app-container` だけでなく `html` / `body` にも専用クラスを付け、root 側の `overflow` と `overscroll` を止める
- 学習画面の通常スクロールは `content-area` に集約し、ブラウザ全体が空白方向へパンする余地を減らす

### 理由
- iPad 系ブラウザでは、内側コンテナを `overflow: hidden` にしていても document 側のパンが残ることがある
- 学習中は問題文・選択肢・手書きメモの操作が中心で、root スクロールを許可する利点より誤操作防止の優先度が高い

## 2026-04-05
### 解答画面ヘッダーの右上操作は `headerActions` で拡張する
- `QuizSessionLayout` の右上領域は AI チャットボタン専用にせず、`headerActions` と既存 AI チャットトグルを並べられる構成にする
- AI チャットトグルの表示条件は既存のまま維持し、ツールボタンは AI パネル表示条件に依存させない
- 解答中の軽量ツールは `SessionToolsLauncher` で一覧・パネル・入力状態をまとめて持つ

### 問題画面電卓の高度計算は独自パーサで扱う
- 問題画面の電卓は `eval` を使わず、括弧・冪乗・平方根・`log`・`ln`・`sin/cos/tan` を扱える再帰下降パーサで式を評価する
- 三角関数は度数法固定とし、`sin(90)=1` になる前提で扱う
- `x²` / `x³` は専用ボタン、一般の冪乗は `xʸ` と `^` で入力する
- 未閉鎖の括弧は表示上は薄い `)` で補助し、評価時は末尾で自動補完する
- バックスペースは文字単位ではなく、`sqrt(` や `sin(`、`^2` / `^3` のような入力単位でまとめて消す
- 式末尾が演算子などで未完成のまま `=` が押された場合はエラーにせず no-op にする
- ボタン配置は科学関数を上段、数字キーを下段、演算子を右列へ寄せた iPad 風レイアウトに寄せる
- 表示上の数値は 3 桁区切りで整形し、内部の評価式にはカンマを入れない
- リサイズ下限は内容量ベースで広めに取り、通常の viewport では全ボタンが枠内に収まる前提を優先する
- `±` は末尾の数値セグメントの符号だけを反転し、`%` は末尾数値を `÷100` に置き換える簡易入力として扱う
- 電卓ウィンドウのリサイズは縦横比固定に切り替え、縮小時は CSS 変数でボタンや文字サイズも比例して縮める
- `√` や `sin` などの前置関数は、直前の値があれば新規に `×` を挟まず、その値を包む形で入力する
- その後の操作性要望に合わせ、電卓のリサイズは再び縦横独立に戻し、縮小時の中身スケールだけを維持する
- さらに調整し、リサイズハンドルは右下のみに限定しつつ、ボタン群は残り高さいっぱいまで広がるレイアウトへ変更する
- 計算結果表示時は iPhone 電卓に寄せて、小さく薄い計算式を結果の上に残す
- 履歴表示行は条件付きで増減させず、最初から高さを確保して表示欄全体の揺れを防ぐ

### ベクトル・行列ラボは front-end only の単独 SVG アプリとして追加する
- ベクトル・行列ラボは API / DB / クラウド同期 / ローカル保存を増やさず、フロントエンドだけで完結する単独アプリとして追加する
- MVP は 2x2 行列と 2 次元平面に限定し、SVG ベースで単位正方形・単位円・基底・ベクトル・像/核を可視化する
- 可視化範囲は fixed 座標にせず auto-fit を前提にし、行列入力や合成結果で主要要素が見切れにくい構成を優先する
- AppLauncher -> Route ラッパ -> 単独コンポーネントの既存構成に合わせ、独立した `linear-algebra-lab-*` クラスで UI を持つ

### 理由（ベクトル・行列ラボ）
- 既存責務を壊さず、学習用アプリを最小差分で追加しやすい
- 外部依存や永続化を増やさず、挙動確認と保守をフロントエンド内で閉じられる
- 今後ほかの単独学習アプリを追加するときも、同じ導線と可視化方針を再利用しやすい

### ベクトル・行列ラボの非モバイル表示は「操作しながらグラフが見える」ことを優先する
- PC / iPad 幅では editor と visualization を縦積みにせず、2 列で同時表示する
- single plot のタブでは visualization を sticky に寄せ、入力カードを縦に追っても平面図を見失いにくくする
- summary は操作と可視化の邪魔をしない位置へ逃がし、まず変化を見られることを優先する
- グラフ中の `Ax` や `Ae1` などは、既存の KaTeX 依存を再利用して数式寄りに表示する
- タッチドラッグは pointer events と広めの hit area、`touch-action: none` でスクロール競合を抑える

### 理由（ベクトル・行列ラボ表示調整）
- この画面は「数値を変えた瞬間に平面図がどう変わるか」を見る用途が中心
- 縦積みレイアウトだと iPad 幅でも入力と可視化を行き来するスクロールが増える
- 数式ラベルの方が、線形代数の記法と図の対応を読み取りやすい
- タブレットやスマホの指操作では、小さい draggable handle だけだとスクロール競合が起きやすい

### ベクトル・行列ラボの記号表示は入力欄も KaTeX でそろえる
- グラフ中の `Ax` や `Ae1` だけでなく、入力欄の `a_{11}` `a_{12}` `x` `y` も既存の KaTeX 依存で描画する
- 数式記号の表示ルールをグラフと入力欄で分けず、同じ見た目で対応づけられることを優先する

### 理由（ベクトル・行列ラボ記号統一）
- 学習用途では、入力している成分と図中の記号が即座に対応して見える方が理解しやすい
- 既存依存で完結でき、追加ライブラリなしで UI の一貫性を上げられる

### iPad 系ブラウザでは SVG 内 `foreignObject` に依存しない
- ベクトル・行列ラボの graph ラベルは、SVG 内の `foreignObject` へ KaTeX HTML を埋め込まず、plot 上の HTML オーバーレイとして重ねる
- タッチ端末では sample vector のドラッグ開始点を handle だけに限定せず、plot 面からも開始できるようにする
- iPad 幅では editor を少し圧縮し、plot を相対的に大きく見せることを優先する

### 理由（iPad 向け graph 調整）
- iPad Safari 系では `foreignObject` の描画や配置が不安定になりやすい
- 学習用途では、ラベルが壊れないことと、指で直接動かしやすいことの方が厳密な埋め込み方式より重要
- touch 操作では、狭い handle だけを狙わせるより plot 面から触れた方が扱いやすい

### 理由
- 学習/暗記の両ルートへ同じ右上 UI を最小差分で差し込める
- 将来ツールを増やしても、AI チャット側の条件分岐を崩さず拡張しやすい
- 問題画面ショートカットとの干渉を、ツール UI ルート単位で管理しやすい
