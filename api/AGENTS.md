# api/AGENTS.md

この配下では、サーバーAPI実装時に以下を厳守する。

## 1. 必須事項
- 認証チェックを行う
- ユーザー所有権チェックを行う
- 内部例外メッセージをそのままクライアントへ返さない
- 認証情報・個人情報をログ出力しない

## 2. 実装ルール
- `api/_http.ts` の `ApiHandlerRequest/ApiHandlerResponse` を使う
- 既存APIのレスポンス形式がある場合、それに合わせる
- 片方のAPIだけ仕様を変えてフロント互換性を壊さない
- DBアクセス時は、対象データがログインユーザーのものか確認する

## 3. DB/マイグレーション
- スキーマ変更は `api/init.ts` に追記する
- `IF NOT EXISTS` を使い冪等にする
- 既存データがある前提で制約追加前の整合性を考える

## 4. セキュリティ
- 500系レスポンスでは抽象化したメッセージを返す
- SQL 実行時は既存の安全な書き方に合わせる
- クエリパラメータや body を信用しすぎない
- 管理者機能や権限付きAPIでは通常ユーザーとの境界を明確にする

## 5. 確認事項
- 未認証時の挙動
- 他人のデータにアクセスした場合の拒否
- 正常系レスポンス
- 入力不足や不正値の扱い
- build 通過
